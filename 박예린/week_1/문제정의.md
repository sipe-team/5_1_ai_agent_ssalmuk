# Week 1: 문제 정의

## 대상 업무

**Claude Code 작업 중 축적된 프로젝트 맥락(Context)의 세션 간 유지 및 재활용**

---

## 현재 문제

Claude Code는 **세션이 끊기면 이전 작업에서 얻은 모든 맥락을 잃는다.**

작업을 거듭할수록 쌓이는 중요한 정보들—레거시 코드의 위험 구간, 반복되는 실수 패턴, 프로젝트 고유의 컨벤션—이 **다음 세션에서 전혀 활용되지 못하고 매번 처음부터 다시 파악**해야 한다.

### 핵심 Pain Point

- 작업 중 발견한 **"이 부분은 조심해야 한다"** 는 인사이트가 세션 종료와 함께 소실
- 다음 세션의 Claude는 동일한 위험 구간을 **다시 탐색하거나 모르고 지나침**
- 코드 리뷰, 유사 기능 개발 등에서 **과거 작업 맥락이 반영되지 않음**
- 사람은 기억하고 있지만 **Claude에게 매번 다시 설명해야 하는 비효율** 발생

### 반복 빈도

**모든 Claude Code 세션** — 특히 레거시 코드베이스, 장기 프로젝트에서 누적 심화

---

## AI로 해결 가능한 포인트

### 1. 세션 중 맥락 자동 포착

터미널 로그와 Claude Code 대화 내역을 분석하여, 단순 작업 기록이 아닌 **"다음에 알아야 할 것"** 을 자동으로 추출

> 예: "auth 모듈의 session.js 237번째 줄—직접 수정 시 토큰 만료 로직 깨짐. 반드시 AuthService 통해서 접근할 것"

### 2. 지식 유형별 분류 및 Obsidian Daily Note 저장

추출한 맥락을 유형별로 구조화하여 날짜별 노트에 누적

- ⚠️ **위험 구간** — 건드리면 문제가 생기는 코드/설정
- 📌 **프로젝트 컨벤션** — 이 프로젝트만의 규칙이나 패턴
- 🔍 **발견한 사실** — 문서화되지 않은 동작 방식
- ✅ **해결 패턴** — 특정 문제를 해결한 접근법

### 3. 유사 키워드 기반 자동 컨텍스트 주입 (RAG)

새 세션에서 특정 파일·모듈·기능을 언급하면, 과거에 기록된 관련 맥락을 **자동으로 검색하여 Claude에게 주입**

- 저장 시: 각 맥락 항목을 벡터 임베딩으로 저장
- 주입 시: 현재 대화의 키워드와 유사도 검색 → 관련 항목 자동 삽입

> 예: "session.js 수정해줘" 입력 시 → 과거 기록 자동 감지 → "⚠️ 이 파일 직접 수정 시 토큰 만료 로직 깨짐" 자동 주입

### 4. 코드 리뷰 시 맥락 활용

PR 또는 코드 리뷰 시 축적된 위험 구간·컨벤션 지식을 기반으로 **과거 맥락이 반영된 리뷰** 제공

## 기대 효과

| 항목 | AS-IS | TO-BE |
| --- | --- | --- |
| 세션 간 맥락 유지 | 세션 종료 시 모든 맥락 소실 | 핵심 인사이트 자동 추출·누적 |
| 위험 구간 인식 | 매번 직접 탐색하거나 실수 후 발견 | 유사 키워드 감지 시 자동 경고 |
| 코드 리뷰 품질 | 현재 코드만 보고 판단 | 프로젝트 역사 기반 판단 |
| 유사 작업 재발 실수 | 같은 실수 반복 가능 | 과거 해결 패턴 자동 참조 |
| 프로젝트 지식 관리 | 개발자 머릿속에만 존재 | Obsidian에 체계적으로 누적 |

## 에이전트 동작 흐름 (초안)

```
[저장 흐름]
Claude Code 세션 종료
        ↓
터미널 로그 + 대화 내역 수집
        ↓
LLM 분석: "다음 세션에서 알아야 할 것"만 추출
        ↓
Obsidian Daily Note에 유형별 저장
        +
벡터 임베딩으로 검색 DB에도 저장

────────────────────────────────
[주입 흐름]
새 Claude Code 세션 — 사용자 입력 발생
        ↓
입력 키워드 → 벡터 유사도 검색
        ↓
관련 과거 맥락 자동 감지
        ↓
Claude 컨텍스트에 자동 주입
(사용자는 별도 조작 없이 Claude가 인지)
```

# Code QnA: 에이전트 역할 설계

## 에이전트 구성

Slack을 통한 소스코드 기반 질의응답 기능을 구현하기 위해, 아래 **4개의 에이전트(노드)**를 설계한다.

### 1. 의도 분류 에이전트 (Intent Classifier Agent)

- **역할**: 사용자의 Slack 메시지를 분석하여 의도(intent)를 판별
- **입력**: 사용자 메시지 (Slack 스레드 컨텍스트 포함)
- **출력**: 의도 분류 결과 (JSON: `{ intent: "question" | "stop" | "other" }`)
- **사용 기술**: LLM (프롬프트 기반 분류, JSON 응답)

### 2. 화면/키워드 추출 에이전트 (Screen Extractor Agent)

- **역할**: 질문에서 화면 이름과 검색 키워드를 추출하여 소스코드 탐색의 단서 마련
- **입력**: 사용자 질문 텍스트
- **출력**: 화면 이름, 키워드 목록 (JSON: `{ screen: string, keywords: string[] }`)
- **사용 기술**: LLM (Few-shot 예시 기반 추출, JSON 응답)

### 3. 화면 매핑 검색 에이전트 (Screen Mapping Agent)

- **역할**: DB의 화면 매핑 테이블에서 추출된 화면/키워드와 유사한 항목을 검색하여 Git 저장소 URL 획득
- **입력**: 화면 이름, 키워드
- **출력**: 매칭된 저장소 URL, 화면 매핑 정보
- **사용 기술**: PostgreSQL 유사도 검색 (Tool 등록)

### 4. 코드 분석 에이전트 (Code Analysis Agent)

- **역할**: Git 저장소의 소스코드를 탐색·분석하여 사용자 질문에 대한 답변 생성
- **입력**: 사용자 질문, 저장소 URL, 화면 매핑 정보
- **출력**: 소스코드 기반 답변 (Markdown)
- **사용 기술**: LangGraph (ReAct 패턴), Git Proxy API (파일 탐색/읽기/검색 Tool)

## 에이전트 간 관계

```
Slack 메시지 수신
  │
  ▼
[1. 의도 분류 에이전트] ──▶ intent 판별
  │                            │
  ├── "stop" ──▶ 작업 중단 응답
  ├── "other" ──▶ 일반 대화 처리
  │
  ▼ "question"
[2. 화면/키워드 추출 에이전트] ──▶ 화면명, 키워드
  │
  ▼
[3. 화면 매핑 검색 에이전트] ──▶ 저장소 URL 획득
  │                                 │
  ├── 매핑 없음 ──▶ 등록 안내 + 어드민 링크 제공
  │
  ▼
[4. 코드 분석 에이전트] ──▶ Git 소스코드 탐색 + 답변 생성
  │
  ▼
Slack 스레드에 최종 답변 댓글
```

## 에이전트별 상세 설계

### 의도 분류 에이전트 상세

| 항목 | 내용 |
| --- | --- |
| 분류 카테고리 | `question` (코드 질문), `stop` (중단 요청), `other` (기타) |
| 중단 인식 패턴 | "그만해", "중지", "취소" 등 |
| 실패 시 기본값 | `other` (안전한 기본값) |
| 호출 방식 | LLM JSON 응답 요청 |

### 화면/키워드 추출 에이전트 상세

| 항목 | 내용 |
| --- | --- |
| 추출 대상 | 화면명 (ex: "주문내역"), 기능 키워드 (ex: "바로주문", "다시담기") |
| 유사도 대응 | 띄어쓰기 차이, 약어 등을 고려한 유연한 추출 |
| 호출 방식 | LLM JSON 응답 요청 |

### 화면 매핑 검색 에이전트 상세

| 항목 | 내용 |
| --- | --- |
| 데이터 소스 | PostgreSQL 화면 매핑 테이블 |
| 검색 방식 | 화면명/키워드 유사도 매칭 |
| 반환 정보 | 저장소 URL, 화면 이름, 관련 경로 |
| 미등록 시 처리 | 어드민 등록 링크 안내 |
| 호출 방식 | ToolRegistry 등록 Tool |

### 코드 분석 에이전트 상세

| 항목 | 내용 |
| --- | --- |
| 실행 방식 | LangGraph ReAct 루프 (agent → tools → agent → ... → END) |
| 사용 Tool | Git Proxy API Tool (프로젝트 검색, 파일 트리 탐색, 파일 읽기, 코드 검색, 브랜치 조회) |
| 무한루프 방지 | `maxIterations` 제한 + `recursionLimit` 설정 |
| 중단 지원 | `AbortSignal`을 통한 실행 중단 |
| 진행 상태 알림 | `onStep` 콜백으로 Slack "답변 생성 중" 메시지 업데이트 |

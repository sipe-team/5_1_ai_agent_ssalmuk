# Week 1: 문제 정의

## 대상 업무

**서비스 장애 대응 및 컴플레인 처리 프로세스**

## 현재 문제

서비스 장애 발생 및 컴플레인 발생 시, 아래 과정을 거치는 데 **평균 60분 이상** 소요됨:

1. 담당자 파악
2. 장애 분류
3. 과거 이력 조회
4. 1차 조치 방법 확인

### 핵심 Pain Point

- 콘텐츠 배포 속도와 업무 효율화  
    - AI Agent 이후 개발 속도는 빨라졌지만 콘텐츠 배포 속도는 빨라지지 않음 
    - 현재 콘텐츠 개발 조직은 마지막에 개발자에게 요청을 해서 앱에 콘텐츠를 적용하는 구조 
    - 개발자의 부하가 가중되며 콘텐츠 배포 속도가 느려짐

### 반복 빈도

**주 3~5회** 

## AI로 해결 가능한 포인트

### 1. 어드민 개발 

- 노코드 개발툴은 오히려 AI 코딩 시대에 병목, 차라리 개발 속도가 더 빠른 상황
- 콘텐츠 배포의 입력 UX 개선

### 2. 콘텐츠 유사도 탐지  (RAG)
- 핵심 콘텐츠인 연애 질문에 대해 의미적 유사도에 대한 VoC가 간간히 인입됨
- 앞으로 유저 세그먼트별, 맞춤 질문을 늘릴 예정인데 VoC가 더 늘어날 걸 염려
- Rag를 통해 시멘틱 검색과 질문 추가 시 유사도 검증을 추가 

### 3. 번역 시스템 내재화
- 핵심 콘텐츠인 연애 질문에 대해 의미적 유사도에 대한 VoC가 간간히 인입됨
- 앞으로 유저 세그먼트별, 맞춤 질문을 늘릴 예정인데 VoC가 더 늘어날 걸 염려
- Rag를 통해 시멘틱 검색과 질문 추가 시 유사도 검증을 추가

### 4. 어드민을 통한 업무 프로세스 일원화
- 현재 흩어진 업무 패턴을 파악 
- 귀찮고 반복되는 업무들에 대한 판단, AI가 자동화할 수 있는 일들에 대한 작업 진행

## 기대 효과

| 항목         | AS-IS                                             | TO-BE                        |
|------------|---------------------------------------------------|------------------------------|
| 어드민 개발     | 개발자 최종 검수로 업무 비대, 콘텐츠 개발 병목                       | 빠른 콘텐츠 배포 가능                 |
| 콘텐츠 유사도 탐지 | 개별 문의, 비체계적                                       | RAG 기반 자동 검색                 |
| 번역 시스템 내재화 | 다국어 처리는 늘 콘텐츠 업무의 주요 병목, 회사의 glossary, 톤앤매너를 유지하기 힘듬 | 다국어 처리에 대한 일관성 및 자동화, 자율성 부여 |
| 어드민을 통한 업무 프로세스 일원화 | 흩어져 정리되지 않은 업무들을 파악하고 정리                          | 정리된 업무들에 대한 AI 사용 방안 탐구      |

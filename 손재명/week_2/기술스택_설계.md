# Week 2: 기술 스택 설계

## 기술 스택 개요

| 영역 | 기술 | 비고 |
| --- | --- | --- |
| 언어 | Python 3.11+ | 에이전트 구현 |
| LLM | OpenAI GPT-5.1 | 장애 분류, 진단, 문서 생성 |
| 에이전트 프레임워크 | LangGraph / CrewAI | 멀티 에이전트 오케스트레이션 |
| 벡터 DB | PostgreSQL + pgvector | 유사 장애 검색 (RAG) |
| 임베딩 | OpenAI Embeddings | 장애 이력 벡터화 |
| 데이터 저장소 | PostgreSQL | 장애 이력, 담당자 정보 |
| 메시지 연동 | Slack API | 장애 접수 및 알림 |
| API 서버 | FastAPI | REST API 제공 |
| 컨테이너 | Docker | 배포 환경 |

## 에이전트별 기술 상세

### 장애 분류 에이전트

- **LLM**: GPT-5.1 (structured output)
- **프롬프트**: Few-shot 예시 기반 분류
- **출력 형식**: JSON (category, severity, keywords)

### 유사 장애 검색 에이전트

- **벡터 DB**: PostgreSQL + pgvector
- **임베딩**: OpenAI text-embedding-3-small
- **검색**: 코사인 유사도, Top-5 반환
- **리랭킹**: LLM 기반 관련성 재평가 (선택)

### 담당자 매칭 에이전트

- **1차**: 규칙 기반 매칭 (카테고리-서비스-담당자 매핑 테이블)
- **2차**: LLM 보조 판단 (모호한 케이스)
- **데이터**: 담당자 DB (이름, 팀, 담당 서비스, 연락처)

### 런북 에이전트

- **기존 런북**: Markdown 파일로 관리
- **자동 생성**: 유사 장애 해결 이력 기반 LLM 생성
- **출력**: 단계별 체크리스트 (Markdown)

### 심층 진단 에이전트

- **LLM**: GPT-5.1 (대화형)
- **입력**: 장애 정보 + 로그 + 유사 사례 + 런북
- **출력**: 원인 분석 리포트, Root Cause 후보

### 포스트모템 에이전트

- **LLM**: GPT-5.1
- **템플릿**: 사내 포스트모템 양식 기반
- **출력**: Markdown 보고서

## 고려 사항

- **비용 최적화**: 단순 작업은 GPT-5.1 mini, 진단/생성은 GPT-5.1 사용
- **프롬프트 관리**: 프롬프트를 코드와 분리하여 별도 파일로 관리
- **데이터 보안**: 장애 로그에 민감 정보 포함 가능 → 마스킹 처리 필요
- **확장성**: 에이전트 추가/수정이 용이한 모듈 구조 설계

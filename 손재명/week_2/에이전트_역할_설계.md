# Week 2: 에이전트 역할 설계

## 에이전트 구성

Week 1에서 정의한 장애 대응 문제를 해결하기 위해, 아래 **6개의 에이전트**를 설계한다.

### 1. 장애 분류 에이전트 (Classifier Agent)

- **역할**: 장애 접수 내용을 분석하여 카테고리와 심각도를 자동 판단
- **입력**: 장애 제목, 설명, 발생 서비스, 에러 로그 (선택)
- **출력**: 카테고리(인프라/앱/DB/네트워크 등), 심각도(Critical/Major/Minor), 키워드
- **사용 기술**: LLM (프롬프트 기반 분류)

### 2. 유사 장애 검색 에이전트 (RAG Agent)

- **역할**: 과거 장애 이력에서 유사 사례를 검색하고 1차 조치 가이드 제공
- **입력**: 분류된 장애 정보 (카테고리, 키워드, 설명)
- **출력**: 유사 장애 Top-K, 과거 해결 방법, 1차 조치 가이드
- **사용 기술**: 벡터 임베딩 + 유사도 검색 (RAG)

### 3. 담당자 매칭 에이전트 (Assignee Agent)

- **역할**: 장애 카테고리/서비스/심각도 기반으로 담당자를 자동 배정
- **입력**: 카테고리, 서비스명, 심각도
- **출력**: 추천 담당자 목록 (주/부), 연락처
- **사용 기술**: 규칙 기반 매칭 + LLM 보조 판단

### 4. 런북 에이전트 (Runbook Agent)

- **역할**: 장애 유형별 대응 절차(플레이북) 제공 및 자동 생성
- **입력**: 장애 카테고리, 유사 장애 해결 이력
- **출력**: 단계별 조치 플레이북, 체크리스트
- **사용 기술**: LLM + RAG (과거 해결 이력 기반 생성)

### 5. 심층 진단 에이전트 (Diagnosis Agent)

- **역할**: 장애 원인을 단계별로 분석하고, 근본 원인(Root Cause) 도출
- **입력**: 장애 정보, 로그, 유사 사례, 런북
- **출력**: 원인 분석 리포트, Root Cause 후보, 추가 확인 사항
- **사용 기술**: LLM (대화형 분석), 로그 파싱

### 6. 포스트모템 에이전트 (Postmortem Agent)

- **역할**: 장애 타임라인과 조치 이력을 기반으로 보고서 초안 자동 생성
- **입력**: 장애 타임라인, 조치 내역, 원인 분석 결과
- **출력**: 포스트모템 보고서 초안 (Markdown)
- **사용 기술**: LLM (템플릿 기반 문서 생성)

```
  │                            │

  │

  │
  ▼
[4. 런북 에이전트] ──▶ 단계별 조치 플레이북
  │
  ▼

  │
  ▼
[6. 포스트모템 에이전트] ──▶ 보고서 초안
```

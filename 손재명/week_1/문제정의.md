# Week 1: 문제 정의

## 대상 업무

**서비스 장애 대응 및 컴플레인 처리 프로세스**

## 현재 문제

서비스 장애 발생 및 컴플레인 발생 시, 아래 과정을 거치는 데 **평균 60분 이상** 소요됨:

1. 담당자 파악
2. 장애 분류
3. 과거 이력 조회
4. 1차 조치 방법 확인

### 핵심 Pain Point

- 여러 부서 담당자에게 **개별 문의**해야 하는 비효율
- 과거 유사 장애 해결 이력이 **체계적으로 관리되지 않음**
- 매번 같은 과정을 **반복**

### 반복 빈도

**주 3~5회** (크고 작은 장애/문의 포함)

## AI로 해결 가능한 포인트

### 1. 자동 장애 분류

LLM이 장애 내용을 분석하여 **카테고리**(인프라/앱/DB/네트워크 등)와 **심각도**를 자동 판단

### 2. 유사 장애 검색 (RAG)

과거 장애 이력을 벡터 임베딩으로 저장하고, 새 장애 발생 시 유사 사례와 해결 방법을 자동 검색하여 **1차 조치 가이드** 제공

### 3. 담당자 자동 매칭

장애 카테고리/서비스/심각도 기반으로 적합한 담당자를 **자동 배정**

### 4. SLA 자동 모니터링

장애 상태별 시간을 자동 추적하고 위반 시 **에스컬레이션 알림**

### 5. 포스트모템 자동 초안 생성

장애 타임라인과 조치 이력을 기반으로 AI가 **보고서 초안 작성**

### 6. 런북(Runbook) 자동 생성 및 관리

장애 유형별 대응 절차를 AI가 자동 생성하고, 과거 해결 이력 기반으로 **단계별 조치 플레이북** 제공

### 7. AI 심층 진단 (Diagnosis)

장애 원인을 단계별로 깊이 분석하고, **근본 원인(Root Cause)** 도출 및 조치 방안을 AI가 대화형으로 안내

### 8. 패턴 탐지

반복 발생하는 장애 패턴을 자동 감지하고, 시간대별/서비스별 발생 빈도를 분석하여 **선제적 예방 조치** 지원

## 기대 효과

| 항목 | AS-IS | TO-BE |
| --- | --- | --- |
| 장애 분류 및 담당자 파악 | 수동, 60분+ | AI 자동 분류, 수 분 내 |
| 과거 유사 장애 조회 | 개별 문의, 비체계적 | RAG 기반 자동 검색 |
| 1차 조치 가이드 | 경험 의존 | 런북/유사 사례 기반 자동 제공 |
| 장애 보고서 작성 | 수동 작성 | AI 초안 자동 생성 |
| 반복 장애 예방 | 사후 대응 | 패턴 탐지 기반 선제 대응 |

# Week 2: 에이전트 역할 설계

## 에이전트 구성

아래 **5개 에이전트**로 구성하되, 각 에이전트의 LLM 호출 여부와 토큰 소비를 명시한다. **LLM을 쓰지 않을 수 있는 단계는 무조건 쓰지 않는다**는 원칙을 적용한다.

---

### 1. 버그 분류 에이전트 (Classifier Agent)

- **역할**: 자연어 버그 리포트를 파싱하여 기능 영역·심각도·영향 레이어·대상 레포를 구조화
- **모델**: Claude Haiku
- **입력**: 버그 리포트 원문 + 사용자 메타데이터
- **출력**: `{ feature_area, severity, affected_layer, affected_repos[], keywords }`

```
feature_area   : "photo_download" | "ai_matching" | "payment" | "auth" | ...
severity       : "critical" | "high" | "medium" | "low"
affected_layer : "frontend" | "api" | "ai_pipeline" | "infra" | "unknown"
affected_repos : ["apiserver"] | ["web"] | ["inference"] | ["apiserver", "web"] | ...

```

**비용 제어:**

- Haiku 사용으로 Sonnet 대비 약 20배 저렴
- 출력은 JSON 최소 필드만 — 설명 텍스트 생성 없음 (zero verbose)
- Few-shot 예시 2개만 포함, 시스템 프롬프트 Prompt Caching 적용
- `affected_layer = unknown AND severity = low` → 즉시 조기 종료, 이후 에이전트 호출 없음

> **멀티 레포 감지**: `affected_repos`가 2개 이상이면 Issue Generator가 각 레포에 동일 타이틀로 Issue를 개별 등록하고 크로스 링크를 자동 첨부한다.

---

### 2. 유사 이슈 검색 에이전트 (Issue RAG Agent)

- **역할**: 기존 GitHub Issues를 벡터 검색하여 중복 감지 및 유사 이슈 컨텍스트 복원
- **모델**: **LLM 호출 없음**
- **입력**: Classifier 출력의 `keywords + feature_area`
- **출력**: `{ duplicate_issue?, similar_issues[], past_fix_summaries[] }`

**비용 제어:**

- pgvector 코사인 유사도 검색만으로 완결 — LLM 비용 0
- 유사도 **0.95 이상** 중복 판정 시 → 파이프라인 즉시 종료, Code RAG·Issue Generator·Suggestion Agent 모두 스킵
- 다음 에이전트에는 이슈 원문 대신 `제목 + 해결방법 한 줄 요약`만 전달 (토큰 절약)
- GitHub Issues 임베딩은 매일 새벽 증분 동기화

---

### 3. 코드 탐색 에이전트 (Code RAG Agent)

- **역할**: 분류 결과를 기반으로 관련 소스 파일·함수·API 엔드포인트·컴포넌트를 레포별로 매핑
- **모델**: Claude Haiku (리랭킹 단계만)
- **입력**: Classifier 출력 + 코드베이스 벡터 인덱스
- **출력**: `{ affected_files[], api_endpoints[], components[], pipeline_stages[] }` (레포별 그룹화)

**비용 제어:**

- 1차 검색은 pgvector만 사용 (LLM 없음), Top-5 후보 추출
- 리랭킹만 Haiku로 처리 → Top-3으로 축소
- LLM에는 파일 전체가 아닌 `함수명 + 시그니처 + docstring + 라인범위`만 전달

```
파일 전체 전달 (X)   → 평균 500~3,000 토큰/파일
함수 메타만 전달 (O) → 평균 30~150 토큰/함수

```

- 연관 파일 0건이면 Suggestion Agent 호출 없이 즉시 종료

---

### 4. Issue 생성 에이전트 (Issue Generator Agent)

- **역할**: 분석 결과를 종합하여 레포별 GitHub Issue 초안 생성 및 등록
- **모델**: Claude Sonnet 4.5
- **입력**: Classifier + Issue RAG + Code RAG 결과 (압축된 형태)
- **출력**: GitHub Issue URL 목록 (레포별 1건씩)

**멀티 레포 처리:**

- `affected_repos` 1개 → 해당 레포에만 Issue 등록
- `affected_repos` 2개 이상 → 동일 타이틀로 각 레포에 Issue 개별 등록

```
예) apiserver#142 본문 하단
Related: web#89 (frontend fix required)

```

**생성 항목:**

- 제목 / 심각도 레이블 / 기능영역 레이블
- 재현 조건 / 영향 범위
- 연관 코드 위치 (파일명 + 라인범위)
- 유사 이슈 링크 (제목 + 요약만)
- AI 추정 원인 초안

**비용 제어:**

- Issue 템플릿을 Prompt Caching prefix로 고정 → 반복 input 토큰 절감
- 입력 데이터는 각 에이전트의 압축 출력만 사용 (원문·파일 원본 미포함)
- Sonnet이지만 컨텍스트가 작아 실질 비용은 낮음

---

### 5. 수정 제안 에이전트 (Suggestion Agent)

- **역할**: Issue 기반으로 파일별 수정 제안을 생성하여 Issue 코멘트로 등록 (실제 코드 변경 없음)
- **모델**: Claude Sonnet 4.5
- **입력**: GitHub Issue + Code RAG Top-3 함수 블록
- **출력**: GitHub Issue 코멘트 (diff 형식 수정 제안)

**조건부 실행 (가장 중요한 비용 제어):**

```
실행 조건 (모두 충족 시에만 호출):
  - severity = critical OR high
  - affected_layer != inference
  - Code RAG 결과 1건 이상

extended thinking 활성화:
  - severity = critical만 해당
  - 나머지는 일반 모드

```

**비용 제어:**

- 함수 블록만 컨텍스트에 포함 (파일 전체 금지)
- 제안 대상 파일 수 Top-3으로 제한
- medium/low 버그는 호출 자체 없음 → Sonnet 비용 0

**제안 코멘트 형식:**

```
## AI 수정 제안

### apiserver/app/api/photos/download/route.ts
- 추정 원인: presigned URL TTL 미설정으로 즉시 만료
- 수정 방향: generatePresignedUrl() 호출 시 expiresIn: 3600 옵션 추가

\`\`\`diff
- const url = generatePresignedUrl(key)
+ const url = generatePresignedUrl(key, { expiresIn: 3600 })
\`\`\`

> 실제 수정 및 PR은 개발자가 직접 진행해주세요.

```

## 에이전트 간 관계

```
버그 리포트 접수 (어드민 게시판)
  │
  ▼
[1. Classifier Agent] — Haiku
  │  출력: feature_area, severity, affected_layer, affected_repos[]
  │
  ├─ unknown + low → 조기 종료 (비용: Haiku 1회)
  │
  ├──────────────────────────┐
  ▼                          ▼
[2. Issue RAG Agent]    [3. Code RAG Agent]
LLM 호출 없음           1차: pgvector, 2차: Haiku 리랭킹
  │                          │
  ├─ 중복(0.95↑) → 종료      ├─ 0건 → Suggestion Agent 스킵
  │                          │
  └──────────┬───────────────┘
             ▼
    [4. Issue Generator Agent] — Sonnet 4.5
    레포별 Issue 등록 + 크로스 링크
             │
             ▼
    [어드민 검토] ── 반려 → 수동 수정
             │ 승인 + (severity=critical/high)
             ▼
    [5. Suggestion Agent] — Sonnet 4.5 (조건부)
    함수 블록 기반 diff 제안 → Issue 코멘트
             │
             ▼
    [개발자 직접 구현 → PR]

```

# Week 2: 기술 스택 설계

## 기술 스택 개요


| 영역         | 기술                            | 비고                        |
| ---------- | ----------------------------- | ------------------------- |
| 언어         | Python 3.12+                  | 에이전트 구현                   |
| LLM        | Claude Sonnet 4.5             | 버그 분류, 코드 분석, Issue/PR 생성 |
| 에이전트 프레임워크 | LangGraph                     | 멀티 에이전트 오케스트레이션           |
| 벡터 DB      | PostgreSQL + pgvector         | 코드베이스 임베딩, 유사 이슈 검색       |
| 임베딩        | OpenAI text-embedding-3-small | 코드/이슈 벡터화                 |
| 코드 인덱싱     | Tree-sitter                   | AST 파싱 → 함수/컴포넌트 추출       |
| GitHub 연동  | PyGithub / GitHub REST API    | Issue 생성, 브랜치, 커밋, PR     |

| 어드민 게시판    | Next.js (기존 프로젝트 /admin)      | 버그 수집 UI                  |
| 컨테이너       | Docker                        | 배포 환경                     |


## 에이전트별 기술 상세

### 버그 분류 에이전트 (Classifier Agent)

- **LLM**: Claude Haiku (structured output / JSON mode)
- **프롬프트**: Few-shot 예시 기반 분류
- **출력 형식**: JSON (feature_area, severity, keywords, affected_layer)

### 코드 탐색 에이전트 (Code RAG Agent)

- **벡터 DB**: PostgreSQL + pgvector
- **임베딩**: OpenAI text-embedding-3-small
- **인덱싱**: Tree-sitter AST 파싱 → 함수명 + docstring + 파일경로 + 라인범위 추출
- **검색**: 코사인 유사도, Top-5 반환
- **리랭킹**: Claude Sonnet 기반 관련성 재평가

### 유사 이슈 검색 에이전트 (Issue RAG Agent)

- **벡터 DB**: PostgreSQL + pgvector
- **동기화**: GitHub Issues 임베딩 저장소 (매일 새벽 갱신)
- **검색**: 코사인 유사도, Top-3 반환

### Issue 생성 에이전트 (Issue Generator Agent)

- **LLM**: Claude Sonnet 4.5
- **출력**: 구조화된 Issue 본문 (Markdown 템플릿 기반)
- **GitHub 연동**: PyGithub / REST API

### 코드 수정 에이전트 (Fix Agent)

- **LLM**: Claude Sonnet 4.5 (extended thinking 옵션)
- **입력**: GitHub Issue + 연관 파일 원본 내용
- **GitHub 연동**: Contents API로 파일 읽기/커밋, PR 생성
- **안전 장치**: 항상 Draft PR로 생성, 어드민 승인 후 머지

## 코드 인덱싱 파이프라인

```
GitHub Action (push 트리거)
  → Tree-sitter로 AST 파싱
  → 함수명 + docstring + 파일경로 + 라인범위 추출
  → text-embedding-3-small으로 벡터화
  → pgvector 저장 (증분 업데이트)
```

## 고려 사항

- **비용 최적화**: 단순 분류는 Haiku, 코드 분석·수정은 Sonnet으로 분리
- **Fix Agent 안전성**: Draft PR + 어드민 승인 필수 / 테스트 파일 자동 수정 제외
- **코드 인덱스 신선도**: GitHub Action (push 트리거)로 증분 업데이트
- **프롬프트 관리**: .yaml 파일로 코드와 분리, 버전 관리
